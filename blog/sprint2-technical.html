CSS Positioning

Positioning elements on a webpage can be a really challenging concept to grasp, but it is super important if you want to be able to have elements that appear outside of the normal flow that is dictated by HTML.

{ position: static }

When you create a website you will likely have an HTML file, with all of the content for your page, that links to a CSS file which provides the style. HTML has baked in styling to all elements including the default position property. As you have likely guessed the default in HTML is [position: static].

See the Pen Static Positioning by Jared (@logansruin) on CodePen.

Inline vs Block

It is worth noting that there are two different default displays, that will impact where an item will appear on the page. By default elements will either have the property [display: inline] or [display: block]. The two most common elements that you will use that illustrate this are <span> and <div>.

{ display: inline }

<span> is an inline element.

“Inline elements are those which only occupy the space bounded by the tags defining the element, instead of breaking the flow of the content” - [https://developer.mozilla.org/en-US/docs/Web/HTML/Inline_elements]

That means if you have a sentence like the one below, to make the words “brown fox” coloured brown you can place them in between <span> tags and apply a style to that <span> without removing breaking the flow of the sentence.

The quick brown fox jumps over the lazy dog

{ display: block }

<div> is a block element.

By default, a block-level element occupies the entire space of its parent element (container), thereby creating a "block." - [https://developer.mozilla.org/en-US/docs/Web/HTML/Block-level_elements]

That means that if we use the same example from above, but wrap the words “brown fox” in <div> tags instead to colour them brown, we expect that they will break the flow of the sentence.

{ position: relative }

The first type of positioning we will look at is relative positioning. Relative positioning allows you to position an element on the page, moving it relative to its original static position in the normal flow.

To do this you need to first change you position to relative:
{ position: relative; }

Then you can use the top, right, bottom and left properties to move the element relative to its original position in the normal flow of the HTML file.

Notice that the elements without class=”relative” applied are still in the same static position unaffected by the change to the elements with relative positioning, even though one of the inline elements now overlaps the other.

{ position: absolute }

The second type of positioning is relative positioning. Absolute is similar to relative positioning as you are still moving an element relative to its original static position in the normal flow. How it differs is that when absolute takes the element out of the normal flow of the HTML which impacts the natural position of elements in the document flow.

To do this you need to first change you position to absolute:
{ position: absolute; }

Just like with relative positioning you use the top, right, bottom and left properties to move the element to a new position.

In the example below if have created a class called absolute, that moves the element 300px from the left of the static position, then applied it to one of the inline elements and one of the block elements.

See the Pen Absolute Positioning by Jared (@logansruin) on CodePen.

As you can see the elements without class=”absolute” applied are no longer in the original static position. The inline element has slipped left into void made by the repositioned inline element. Less noticeably the block element with the class applied has shifted down another 10px which I assume has something to do with how vertical margins are applied differently when taken out of the normal flow.

{ position: fixed }

The third and final type of positioning that I will review is fixed positioning. Fixed positioning is similar to absolute and relative positioning because you can give directions to move the element relative to its original static position in the normal flow. Also like absolute positioning, but different to relative, it takes the element out of the normal flow of the HTML which impacts the natural position of elements in the document flow.

It differs from both relative and absolute by fixing them into the allocated position on the screen.

To do this you need to first change you position to fixed:
{ position: fixed; }

Just like with relative and absolute positioning you use the top, right, bottom and left properties to move the element to a new position.
